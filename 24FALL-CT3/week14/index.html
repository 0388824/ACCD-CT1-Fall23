Digital Materialities

Fake 3D Roads

The road is created using a perspective effect, where its width gradually decreases as it moves toward the horizon. This is achieved by adjusting the road's width based on the uv.y coordinate. As the uv.y value changes, the road narrows from a wider base near the bottom of the screen to a smaller width near the top. The road's position is determined by comparing the pixel's uv.x value with the calculated road width.

Random Generated Patterns

These five visuals are created using a combination of randomness, time-based changes, mathematical shapes, and color blending. Random values add variation, time creates movement, and math functions like sine waves and polar coordinates shape the patterns. This approach allows simple inputs, like positions and angles, to generate complex, ever-changing visuals. Each design – whether it's a swirling vortex, flowing waves, or scattered stars – shares this core method of continuous transformation and dynamic generation.

3D Models

These five 3D shapes – a sphere, a cube, a torus, a cylinder, and a cone – are rendered purely through code-driven math. Each form is defined by a signed distance function (SDF) that describes how far any point in space is from its surface. By casting rays pixel by pixel and incrementally "marching" forward until hitting a shape or the ground plane, the fragment shader determines what to draw. Normals are computed on the fly by sampling nearby points, enabling dynamic lighting without stored models.
